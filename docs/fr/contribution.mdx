# Contribuer à Agent Zero

Les contributions pour améliorer Agent Zero sont très bienvenues ! Ce guide décrit comment contribuer au code, à la documentation ou à d'autres améliorations.

## Commencer

1. **Forker le dépôt :** Forkez le dépôt Agent Zero sur GitHub.
2. **Cloner votre fork :** Clonez votre dépôt forké sur votre machine locale.
3. **Créer une branche :** Créez une nouvelle branche pour vos changements. Utilisez un nom descriptif qui reflète le but de votre contribution (par exemple, `fix-memory-leak`, `add-search-tool`, `improve-docs`).

## Faire des modifications

* **Style de code :** Suivez le style de code existant. Agent Zero suit généralement les conventions PEP 8.
* **Documentation :** Mettez à jour la documentation si vos changements affectent la fonctionnalité utilisateur. La documentation est écrite en Markdown.
* **Messages de commit :** Écrivez des messages de commit clairs et concis qui expliquent le but de vos changements.

## Soumettre une Pull Request

1. **Pousser votre branche :** Poussez votre branche vers votre dépôt forké sur GitHub.
2. **Créer une Pull Request :** Créez une pull request de votre branche vers la branche appropriée dans le dépôt principal Agent Zero.
   * Ciblez la branche `development`.
3. **Fournir des détails :** Dans la description de votre pull request, expliquez clairement le but et la portée de vos changements. Incluez le contexte pertinent, les résultats de test et toute autre information qui pourrait être utile pour les réviseurs.
4. **Répondre aux commentaires :** Soyez réactif aux commentaires de la communauté. Nous aimons les changements, mais nous aimons aussi en discuter !

## Stack de documentation

- La documentation est construite en utilisant Markdown. Nous apprécions vos contributions même si vous ne connaissez pas Markdown, et nous attendons avec impatience d'améliorer Agent Zero pour le bénéfice de tous.
