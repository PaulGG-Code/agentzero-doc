# Contribuire ad Agent Zero

I contributi per migliorare Agent Zero sono molto benvenuti! Questa guida delinea come contribuire con codice, documentazione o altri miglioramenti.

## Iniziare

1. **Fork del Repository:** Fai un fork del repository Agent Zero su GitHub.
2. **Clona il tuo Fork:** Clona il tuo repository forkato sulla tua macchina locale.
3. **Crea un Branch:** Crea un nuovo branch per le tue modifiche. Usa un nome descrittivo che rifletta lo scopo del tuo contributo (es. `fix-memory-leak`, `add-search-tool`, `improve-docs`).

## Fare Modifiche

- **Stile Codice:** Segui lo stile di codice esistente. Agent Zero generalmente segue le convenzioni PEP 8.
- **Documentazione:** Aggiorna la documentazione se le tue modifiche influenzano funzionalità visibili all'utente. La documentazione è scritta in Markdown.
- **Messaggi Commit:** Scrivi messaggi commit chiari e concisi che spieghino lo scopo delle tue modifiche.

## Inviare una Pull Request

1. **Push del tuo Branch:** Fai push del tuo branch al tuo repository forkato su GitHub.
2. **Crea una Pull Request:** Crea una pull request dal tuo branch al branch appropriato nel repository principale Agent Zero.
   - Mira al branch `development`.
3. **Fornisci Dettagli:** Nella descrizione della tua pull request, spiega chiaramente lo scopo e la portata delle tue modifiche. Includi contesto rilevante, risultati dei test e qualsiasi altra informazione che potrebbe essere utile per i revisori.
4. **Affronta Feedback:** Sii reattivo al feedback dalla community. Amiamo i cambiamenti, ma amiamo anche discuterli!

## Stack Documentazione

- La documentazione è costruita usando Markdown. Apprezziamo i tuoi contributi anche se non conosci Markdown, e non vediamo l'ora di migliorare Agent Zero per il beneficio di tutti.
