# Contribuir a Agent Zero

¡Las contribuciones para mejorar Agent Zero son muy bienvenidas! Esta guía describe cómo contribuir código, documentación u otras mejoras.

## Comenzar

1. **Hacer Fork del Repositorio:** Haz fork del repositorio de Agent Zero en GitHub.
2. **Clonar tu Fork:** Clona tu repositorio fork a tu máquina local.
3. **Crear una Rama:** Crea una nueva rama para tus cambios. Usa un nombre descriptivo que refleje el propósito de tu contribución (ej. `fix-memory-leak`, `add-search-tool`, `improve-docs`).

## Hacer Cambios

- **Estilo de Código:** Sigue el estilo de código existente. Agent Zero generalmente sigue las convenciones PEP 8.
- **Documentación:** Actualiza la documentación si tus cambios afectan la funcionalidad visible al usuario. La documentación está escrita en Markdown.
- **Mensajes de Commit:** Escribe mensajes de commit claros y concisos que expliquen el propósito de tus cambios.

## Enviar un Pull Request

1. **Hacer Push de tu Rama:** Haz push de tu rama a tu repositorio fork en GitHub.
2. **Crear un Pull Request:** Crea un pull request desde tu rama a la rama apropiada en el repositorio principal de Agent Zero.
   - Apunta a la rama `development`.
3. **Proporcionar Detalles:** En la descripción de tu pull request, explica claramente el propósito y alcance de tus cambios. Incluye contexto relevante, resultados de tests y cualquier otra información que pueda ser útil para los revisores.
4. **Abordar Feedback:** Sé receptivo al feedback de la comunidad. ¡Nos encanta los cambios, pero también nos encanta discutirlos!

## Stack de Documentación

- La documentación está construida usando Markdown. Apreciamos tus contribuciones incluso si no conoces Markdown, y esperamos mejorar Agent Zero para el beneficio de todos.
